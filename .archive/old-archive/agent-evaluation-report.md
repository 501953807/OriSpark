# OriStudio v3 多角色评估报告

> **日期**: 2026-06-12 | **评估角色**: 9位 (6类创作者 + 1新手 + 1架构师 + 1律师)
> **评估方式**: 每位角色基于 docs/modules-v3/ 全套设计文档独立评估

---

## 一、各角色核心评价

### 1. 林山海 (插画师/AIGC艺术家)

**综合评分: 60/100**

**关键肯定**: 拖拽导入+自动SHA-256方向对; 版本管理阶段标签贴合真实绘画流程; 60+品类分类诚实; 6种平台风格AI文案思路正确

**尖锐批评**:
1. **手动填synopsis/completion_date/creation_tool → 让我做 Excel 文员** — 应从文件名/EXIF自动抓取
2. **SHA-256哈希值在前端展示 → 那是给程序员看的** — 换成"已存证 ✅ 2026-06-12"
3. **没有批量导入已有文件夹结构** — 不支持文件夹递归读取、按文件夹名自动创建项目组, 第一天迁移成本太高就不会用
4. **Canvas效果预览严重低估难度** — T恤有褶皱/弧度/布料纹理, 2D drawImage做不了真实mockup
5. **POD对接是"假对接"** — 只是URL记录而非API级同步。没有API就不叫对接
6. **手动收入登记** — 又让做账。应能导入支付宝/微信/POD平台CSV
7. **发布 → 跳转到平台手动操作** — 把闹钟改了个名字
8. **AI文案需要个人风格学习** — 没有历史文案学习/用户风格存档
9. **没有商业约稿流程** — 40%收入来自约稿却完全没有管理
10. **没有手机端** — 核心缺失

**最需要的3个功能**: (1)真实照片级产品效果预览 (2)一键多渠道POD同步 (3)社交媒体数据自动汇总

### 2. 赵一鸣 (短视频/动画创作者)

**综合评分: 系统对视频创作者是"硬塞进来"的**

**尖锐批评**:
1. **视频不是单一文件** — 是一个项目包(素材+工程文件+调色+字幕+字体)。只接受MP4/MOV等于告诉插画师"你的作品就是导出的JPG, 没有PSD图层"
2. **变现路径几乎空壳** — 50%收入的平台激励完全无追踪; 30%的品牌商单零覆盖
3. **品牌商单工作流缺失**: brief→方案→脚本→制作→反馈→交付→结算, 每步都有管理需求
4. **工程文件不被支持**: .prproj/.aep/.drp 完全无法导入
5. **视频指纹只提了概念** — 无perceptual hashing设计, SHA-256重编码就变
6. **字幕管理缺失** — .srt/.ass、双语字幕、字幕样式完全没有
7. **协作审片缺失** — 无Frame.io式时间码标注评论
8. **横竖屏版本管理缺失** — 同一视频需16:9/9:16/1:1不同构图
9. **大文件处理严重不足** — "100MB异步"对视频是门槛都不到, 版本快照会导致硬盘爆炸
10. **缩略图"提取首帧"** — 首帧大概率全黑, 应提取中间帧

**结论**: 系统是优秀的插画师全链路工具, 视频支持是被承诺但未兑现的附加功能

### 3. 周巧手 (手工艺人/文创设计师)

**综合评分: 系统是为"数字文件→印到产品上"设计的, 手工艺人是"先做实物→拍照→照片上线"**

**尖锐批评**:
1. **没有物理原件概念** — 系统closest concept是"work with file_type=image", 但物理原件是主实体, 照片是衍生
2. **没有原料库存管理**: 5种粘土存量、30+釉料化学品、保质期、补货阈值
3. **没有生产批次**: 批次号、投入数量、过程阶段、缺陷率、产出统计
4. **没有质检分级**: A/B/C级、缺陷照片、根本原因分析
5. **没有成本核算**: 材料+人工+窑炉电费+包装+物流=真实单位成本
6. **Etsy只在seed data里出现** — 没有Etsy API网关(listing/订单/库存/物流/消息)
7. **50%收入来自Etsy但系统完全不管Etsy** — 不可接受
8. **产品摄影工作流缺失** — 需要白平衡校正/背景去除/多角度/一致灯光
9. **"按需制造"是给"发设计稿给工厂"设计的** — 不是给"我用手做东西"设计的

**结论**: 不会从 Etsy + Google Sheets 切换到 OriStudio

### 4. 陈摄影 (摄影师/视觉创作者)

**综合评分: 1.9/10 — 不适合专业摄影师使用**

**逐项评分**: 工作流理解3/10 | RAW支持1/10 | EXIF利用4/10 | 商业转化2/10 | 水印1/10 | 图库集成0/10 | 批量操作2/10

**尖锐批评**:
1. **RAW支持设计稿有, 代码一行都没实现** — ALLOWED_EXTENSIONS没有RAW/CR2/NEF/ARW/DNG, detect_file_type()落到"other"
2. **过程阶段是插画师的** — 摄影师是: 拍摄→导入→选片(Culling)→基础调整→精细调色→局部修饰→导出(多尺寸/多格式)→交付
3. **EXIF提取了, 浪费了** — 按镜头/光圈/焦段/ISO筛选完全做不到。没有地图视图浏览GPS位置
4. **水印后端代码零实现** — watermark_service.py不存在。无批量水印/平铺水印/差异化水印强度
5. **图库集成纯纸面** — 无Shutterstock/500px/图虫API适配器。发布流程写着"手动执行"
6. **批量导入无预设元数据** — 1000张照片需逐一操作。缺少选片标记/批量导出/批量重命名
7. **每张RAW 50MB+, 无存储策略** — 一个拍摄64GB数据, 系统无磁盘配额/清理机制

**如果想服务摄影师, 至少需要**: rawpy/libraw RAW解码、选片(Culling)模式、批量元数据模板、xmp sidecar管理、真正的后端水印服务、EXIF驱动搜索面板、至少对接一个图库API

### 5. 苏旋律 (音乐人/音频创作者)

**综合评分: 系统对音乐产业一无所知**

**尖锐批评**:
1. **PRO注册、机械版税、ISRC码** — 需求文档提到了, 模块设计全部缺失
2. **没有ISRC码管理** — DistroKid拒绝没有ISRC的上传
3. **没有Split Sheets** — 合作者权益分配完全不存在, 只有一个author_name字段
4. **没有Master vs Publishing权利区分** — 音乐行业最基础的法律区分, 系统用单个rights JSON覆盖
5. **没有Sample Clearance追踪** — 采样授权来源+商业用途许可
6. **变现引擎对音乐人几乎不相关** — 5条路径只有众筹部分适用
7. **Product Designer选音频文件会怎样？设计没有回答**
8. **没有专辑/EP/Single概念** — 曲目排序/统一定价/发行日期/封面3000×3000规范
9. **分发平台不包含任何音乐平台** — Spotify/QQ音乐/网易云/Apple Music不在8个平台列表中
10. **音频指纹只是占位符** — AcoustID有速率限制, YouTube Content ID需合作伙伴资格

**结论**: 需要一个平行的音乐工作流, 不能把音乐"拧"进视觉架构

### 6. 文墨 (文字作者/编剧)

**综合评分: 系统是由视觉创作者为视觉创作者设计的, 文字作者是"贴上去的"**

**尖锐批评**:
1. **没有章节概念** — "一个work就是一个文件"。写作者需要: 卷→章→草稿/设定/反馈/批注的层级结构
2. **7步产品向导对文字作者完全不可用** — Step4 Canvas效果预览无法处理文字产品
3. **抄袭检测只有"手动"两个字** — 侵权监测完全以图搜图导向
4. **没有EPUB/PDF排版导出** — 无Pandoc管道/章节合并/目录生成
5. **版本管理全文件快照不适合** — 需要章节级修订跟踪/编辑反馈/beta reader意见
6. **权利管理"一个下拉框"覆盖不了** — 翻译权/影视权/连载权/有声书权/游戏改编权/地域/期限/阶梯版税全部缺失
7. **分发平台不包含写作平台** — 需求写了起点/晋江/KDP, 分发中心设计里一个字没出现

**结论**: 要么诚实收缩为视觉创作者工具, 要么投入从数据模型层面重新设计

### 7. 小明 (新手用户/入门3个月)

**综合评分: 功能太多、术语太难、路径不清晰**

**尖锐批评**:
1. **"7个模块？那是7个我不认识的图标"** — 从画完到保护到变现的路径不直观
2. **9个专业术语只能理解3个** — SHA-256/EXIF/DPI/POD/C2PA/WIPO/尼斯分类/洛迦诺/Canvas mockup
3. **三个独立向导=三种困惑路径** — 7步+5步+6步=18步, 不知道从哪个开始
4. **一打开全是空图表** — 空收入/空分析/空合作伙伴 → 感觉自己失败了还没开始
5. **如果第一步是填法律表格, 立即卸载**
6. **没有教程或入门引导 → 立即卸载**
7. **看起来像企业财务软件 → 立即卸载**
8. **需要"神奇按钮"** — 一键导入→一键保护→一键变产品, 隐藏所有复杂性

**一句话**: "如果像Canva(友好/多彩/简单)→留下。如果像SAP→30秒卸载。商业功能只在我准备好时才出现。"

### 8. 架构师 (15年SaaS/创作工具经验)

**核心发现**:
- **代码比文档更完整**: 实际49个模型类, 230+端点, 35KB的dict_seed.py — 文档只写了33表和145端点
- **缺失数据模型**: Order/Partner/OrderPayment表不存在于ORM中, 但制造/众筹工作流需要
- **文档与代码不匹配**: models/publish.py包含Product(属模块4)和publishing模型, 文件命名没跟上v3模块边界
- **SQLite WAL 对单用户桌面工具完全够用**, PostgreSQL迁移不是config change而是重大工程
- **最大的架构错误**: 试图用一个统一数据模型服务6种根本不同的创作者, 导致每个类型都照顾到但都不深入
- **建议**: v1聚焦1个创作者类型(插画师), 每个新增类型作为major version升级

**可行性评级**:
- ✅ 可构建: 多格式导入、SHA-256存证、PDF证书、字典中心、手动登记CRUD、产品目录种子数据
- ⚠️ 中等难度: Canvas mockup(扁平叠加可行, 照片级需CV)、以图搜图(API有但额度低)、6辖区费用计算(数据维护噩梦)
- ❌ 愿景级: 视频/音频指纹(不是feature是产品)、C2PA内容凭证(如半实现会有虚假安全感)

### 9. 法律顾问 (12年IP/数字合规经验)

**最大法律风险**: 商标分类推荐 + 可注册性评估 + CNIPA自动提交 = 自动商标代理。正是中国律师法禁止的"无证从事法律服务"。

**风险矩阵**:

| 风险 | 严重度 | 可能性 | 优先级 |
|------|--------|--------|--------|
| 无证从事法律服务(中国) | 关键 | 中 | **P0** |
| POD平台IP泄露/间接侵权 | 高 | 高 | **P0** |
| 跨境数据传输(PIPL/GDPR) | 高 | 中 | P1 |
| AI内容标注不合规 | 中高 | 高 | P1 |
| 侵权监测爬虫合法性 | 中 | 中高 | P2 |
| 尼斯分类建议构成法律意见 | 中 | 高 | P2 |
| 免责声明不充分 | 中 | 高 | P2 |
| 未成年人行为能力 | 低中 | 低 | P3 |

**必须补全的7项免责声明**: 不构成律师-客户关系 / 不构成法律建议 / 不保证注册成功 / POD平台条款警告(当前完全缺失) / AI内容标注要求 / 侵权监测局限性 / 司法管辖区限制

**关键合规建议**:
- 尼斯分类→多类可能性+置信度, 而非单一推荐
- CNIPA提交→"推荐律师审核"步骤, 不可绕过
- POD上传前→强制显示平台IP条款摘要
- 首次启动→强制确认免责声明

## 四、必须修改的关键设计缺陷

### 🔴 P0 — 致命级 (不可上线)

1. **重新设计产品设计器** — Canvas mockup达不到商业级。要么接入Printful mockup API, 要么使用照片级模板
2. **砍掉"假POD对接"** — 没有API就不叫对接, 明确声明手动上架+URL记录, 诚实面对能力边界
3. **增加批量导入文件夹** — 迁移成本问题, 第一天不用=永远不会用
4. **补全法律合规**: 7项免责声明 + POD条款警告 + AI标注 + 尼斯分类多选+置信度 + CNIPA提交前律师审核步骤
5. **聚焦一个创作者类型** — v1只做插画师/AIGC艺术家, 其他类型明确标为"规划中"

### 🟡 P1 — 严重级

6. **非视觉类型数据模型**: 专辑/曲目(音乐)、章节/卷(文字)、工程项目包(视频) — 至少设计预留
7. **补充分发平台**: Spotify/QQ音乐(音乐)、起点/晋江/KDP(文字)、Etsy(手工) — 至少加入平台列表
8. **增加入门引导** — 首次启动按创作者类型引导, 支持选择"我刚开始/已有作品库/专业创作者"
9. **补全缺失数据模型**: Order/Partner/OrderPayment表
10. **修正文档**: 49模型→文档, 230+端点→文档

### 🟢 P2 — 需要级

11. **移动端** — 至少要有响应式Web
12. **EXIF搜索过滤面板** — 按镜头/光圈/焦段/ISO
13. **自动元数据抓取** — 文件名→日期, EXIF→参数, 不要手动填
14. **水印服务** — 后端实现, 支持批量+平铺+差异化强度
15. **选片(Culling)模式** — 批量浏览+入选/待定/淘汰标记
16. **RAW格式支持** — rawpy/libraw集成

## 五、建议的修正路径

```
当前设计(v3): 试图服务6种创作者 → 每种都照顾到, 每种都浅
         ↓
修正方向:
  第一步: v1聚焦插画师/AIGC艺术家 → 做到极致
  第二步: v2加摄影师 → 相似的视觉工作流, 增加RAW支持
  第三步: v3加视频 → 较大差异, 需独立数据模型
  第四步: v4+ → 音乐/文字/手工 → 每类都有独立工作流

短期(v1): 砍掉音乐/文字/视频/手工的"专属功能"承诺
         诚实标记为"规划中", 专注把插画师的体验做到Canva级别
```
