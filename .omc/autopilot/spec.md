# OriStudio v3 UX/UI 重设计方案

> **日期**: 2026-06-13 | **状态**: Phase 0 完成

## 一、诊断结果

### 核心发现：业务链断裂，模块无入口

| docs/modules-v3 | 菜单 | 问题 |
|---|---|---|
| [6] 经营管理中心 | **无菜单** | 收入追踪、合作伙伴、订单、通知零入口 |
| [3] IP登记工作站 | 📋 IP登记 | 每次全屏免责弹窗阻断操作，6辖区纵向堆砌 |
| [2] 权利保护中心 | 🔒存证确权 + 🛡️侵权监测 | 拆成两个菜单，业务上应合并 |
| [5] 内容分发中心 | 📊 产品数据 | 命名不一致 |
| [4] 商业转化引擎 | 💰 创作变现 | 8个Tab混在一起，收入追踪被埋 |

### 缺失的功能入口

| 模块 | 功能 | 当前入口 |
|------|------|---------|
| [6] | 收入追踪 + CSV导入 | SupplyView 第8个Tab(找不到) |
| [6] | 合作伙伴管理 (Partner模型已有) | SupplyView 第7个Tab |
| [6] | 订单管理 (Order模型已有) | SupplyView Tab内 |
| [6] | 通知中心 | Topbar角落 |
| [6] | 数据分析看板 | Dashboard(仅2个卡片) |
| [1] | 项目管理 (Project) | 侧边栏无入口 |
| [7] | Onboarding完成状态 | 无持久入口 |

## 二、UX重设计方案

### 2.1 新侧边栏结构

```
📊 工作台 (Dashboard — 业务总览)
───────────────────────────── (核心业务链)
🎨 创意资产 (Module 1 — 作品导入/管理/版本/标签)
🔒 权利保护 (Module 2 — 存证+监测 单页面两Tab)
📋 IP 登记 (Module 3 — 交互式向导重构)
💰 商业转化 (Module 4 — 产品设计器+POD+众筹+授权)
📱 内容分发 (Module 5 — 发布+AI文案+影响力)
📈 经营管理 (Module 6 — 收入+合作伙伴+订单+通知)
─────────────────────────────
⚙️ 系统设置 (Module 7 — 合并偏好+对接+Onboarding)
```

### 2.2 关键页面重设计

**📈 经营管理中心 (BusinessView — 全新页面)**
- 聚合仪表盘: 收入/合作伙伴/订单/通知汇总卡片
- 收入追踪Tab: 手动登记+CSV导入+ECharts图表+来源类型(约稿/POD/授权/版税/其他)
- 合作伙伴Tab: Partner CRUD + 类型筛选(供应商/工厂/POD平台/客户)
- 订单Tab: Order CRUD + 状态流转
- 通知Tab: 分类筛选+关联跳转
- 数据分析Tab: 收入趋势+产品分布

**📋 IP 登记 (IprView — 完全重做)**
- 首次: 半屏Banner(非Modal)，接受后persist
- 操作区: IP类型选择器 → 辖区切换 → 指引卡片
- 指引卡片: 所需材料+办理流程+费用+官方链接
- 智能助手: 描述创作→推荐尼斯分类+置信度
- 我的登记: CRUD+状态追踪
- 律师审核步骤: 选项A/B/C, 前置条件(联系律师/自行承担/暂不提交)

**📱 内容分发 (PublishView — 重命名)**
- 菜单名: "产品数据" → "内容分发"
- 发布向导(WizardStepper): 选择内容→选择平台→AI文案→图片处理→确认发布

### 2.3 业务链指示器

顶部业务流程条(所有核心页面共享):
```
上传作品 → 存证确权 → IP登记 → 产品设计 → 分发推广 → 收入追踪
✅        ✅        ⬜       ⬜       ⬜       ⬜
```

### 2.4 全局UX改进

| 问题 | 当前 | 改进 |
|------|------|------|
| 免责声明 | 每次全屏Modal | 首次半屏Banner, acceptance记录 |
| 信息堆砌 | 6辖区纵向罗列 | Tab切换, 单辖区清晰 |
| 功能入口 | 经营管理6功能无入口 | 独立页面, 聚合展示 |
| 菜单命名 | "产品数据"≠"内容分发" | 统一命名 |
| 业务链 | 无上下文 | 顶部业务流指示器 |

## 三、实施计划

### Phase 1: 菜单路由重构 (第1-2天)
1. 修改 AppSidebar — 新菜单结构: "权利保护"合并, "内容分发"重命名, "经营管理"新增
2. 修改 router — 新增 `/app/business` 路由
3. 修改 AppTopbar — 补全 pageTitles

### Phase 2: 经营管理中心 (第3-4天)
4. 新建 BusinessView.vue — 聚合收入+合作伙伴+订单+通知+分析
5. 新建 useBusinessStore.ts + api/business.ts
6. 从 SupplyView 移除收入/合作伙伴Tab
7. Dashboard 增加业务总览卡片

### Phase 3: IP登记重设计 (第5-6天)
8. 重写 IprView — 交互式向导替代纵向罗列
9. 修改免责声明逻辑 — first-visit Banner + persistence
10. 新建组件: IpTypeSelector, JurisdictionGuide, SmartAssistant

### Phase 4: 业务链打通 (第7-8天)
11. 新建 BusinessChainBar — 顶部业务流指示器
12. 各页面增加跨模块快捷跳转
13. 全流程串联测试
