OriStudio 个人创作者助手 完整建设方案

系统设计说明书 · 基于7路专家评审完善 · 2026年6月10日

1. 设计理念

一句话定义：一个在个人电脑上运行的创作者全链路助手工具，帮助没有经验的独立创作者快速完成作品管理、存证确权、侵权监测、知识产权登记、供应链协作和产品落地的完整业务闭环。

核心原则

原则说明设计体现
创作者主权数据永远在创作者的电脑上本地SQLite + 本地存储，不上传创作者原始文件
不重复造轮子存证/商标/搜索等通过API接入现有平台用户扫码自助付费，工具只做中间桥梁
渐进式复杂度新手只看得到基础功能，高级功能可展开默认界面仅展示核心流程，高级设置在设置页
离线优先无网络时仍可使用核心功能本地哈希计算 + 本地数据库完全离线可用
易于移植一键打包迁移到任何电脑Docker + PyInstaller + pip 三种分发方式
📋 评审改进及用户修正汇总
1️⃣ 电商发布：非CSV导出！改为一键扫码登录+Playwright浏览器自动化+开放平台OAuth混合方案，实现真一键发布到淘宝/小红书/抖音
2️⃣ 商标注册：用户明确费用是用户扫码直接支付给国家平台，系统只是技术桥梁。保留表单预填+提交辅助功能，删除的是"成功率预测"而非"提交辅助"
3️⃣ 存证/注册只对接国家/大平台：版权家(国家版权局DCI)、至信链(司法链)、商标局电子申请系统、中国版权保护中心
4️⃣ 架构模式：B/S模型优先（Python后端+Vue前端本地运行），后期支持PWA安装到桌面/可选服务器部署
5️⃣ 供应链大幅精简：砍掉库存管理/甘特图，保留轻量工厂协作记录
6️⃣ 增加维权闭环：侵权发现后→投诉材料生成→证据包导出→律师函模板

6. 存证确权模块

📋 评审改进
法律评审建议：存证平台选择界面需标注各平台司法效力级别 （版权家DCI国家版权局体系 > 至信链司法链 > 蚂蚁链商业链）
技术评审建议：哈希计算和缩略图生成需异步执行，不阻塞用户操作
用户强调：所有存证必须对接国家/大平台才有真实效力

存证平台对接策略

平台性质对接方式费用司法效力
版权家（国家版权局DCI体系）国家级HTTP API + 用户扫码自付¥1-5/次高（国家版权局认可）
至信链（腾讯/互联网法院）司法链HTTP API + 用户扫码自付¥1-1.5/次高（法院采信）
蚂蚁链商业链HTTP API + 用户扫码自付¥0.5/次中（需额外司法认定）
国家知识产权局电子申请国家级引导用户自助办理（附材料清单和填写参考）官方标准最高

所有存证费用由用户通过二维码扫码直接支付给平台，系统不碰资金。本地留存存证记录和证书供用户查询。

功能清单

子功能优先级详细描述
本地哈希计算P0对作品文件进行SHA-256哈希计算（异步任务），生成哈希值+时间戳+文件属性记录，存入本地SQLite。离线可用
多平台存证对接P0内置3个存证平台：①版权家(¥1-5/次,司法采信) ②蚂蚁链(¥0.5/次,联盟链) ③至信链(¥1-1.5/次,腾讯)。用户选择平台→弹窗二维码→扫码自助付费→平台返回存证ID→存入本地
C2PA元数据嵌入P1将创作过程数据（软件/时间/版本）嵌入作品文件（支持PNG/JPG/WebP/MP4），开源c2patool实现
PDF证书生成P0生成存证证书PDF（作品缩略图+哈希值+时间戳+存证平台标识+验证二维码），用户可打印/分享
存证记录管理P0时间线视图展示所有存证记录；状态标识（未存证/已存证本地/已存证平台/待验证）；一键批量存证
验证入口P1消费者/第三方可通过OriStudio验证页面输入哈希值或扫描二维码验证真伪（读取存证平台数据）

存证操作流程

作品库 → 选中作品 → 点击"存证" → Step 1: 本地SHA-256哈希计算（异步，大文件不卡UI） → Step 2: 选择存证平台（版权家/蚂蚁链/至信链）→ 显示费用和司法效力说明 → Step 3: 弹出二维码 → 用户手机扫码 → 自助付费 → Step 4: 等待平台确认（异步轮询状态） → Step 5: 生成PDF证书 + 更新存证记录 → 完成！证书可导出/分享，验真二维码可嵌入作品展示页

功能清单

子功能优先级详细描述
以图搜图P0选中作品→调用百度识图免费版(100次/天)或Google Vision(1000次/月免费)→自动比对搜索结果→标记高相似度结果

8. IP登记模块（引导型）

法律澄清（用户确认）：
系统只是技术桥梁——表单预填+API提交辅助，不收费、不作法律判断。
费用由用户扫码直接支付给国家平台（商标局/版权局）。这不构成商标代理。
保留功能：商标查询引导、多品类规划建议、表单预填、API提交辅助
删除功能：「注册成功概率预测」（因为无法准确评估，非法律原因）

功能清单

子功能优先级详细描述
版权登记引导P0根据作品类型（美术/文字/音乐/软件）引导用户准备对应材料（作品样本+创作说明+权利保证书），提供材料清单和填写参考，链接跳转国家版权局登记平台
商标查询引导P0引导用户到商标局官网（http://sbj.cnipa.gov.cn）进行近似查询；根据作品类型推荐注册类目（核心类目+关联类目）；提供查询结果记录表
多品类规划P1根据用户选择的产品类型自动推荐商标注册类目组合（如做文创→16类纸品+25类服装+28类玩具+35类广告）；生成类目规划表供用户参考
进度看板P0手动记录商标/版权申请的提交日期、申请号、状态、预计完成时间；设置续展/异议提醒（日历通知）
续展提醒P1商标续展期（到期前12个月）自动提醒；版权保护期到期提醒

9. 供应链协作模块（轻量版）

评审共识：供应链要轻、要真、要能用
供应链评审指出：工厂无动力维护创作者本地工具中的数据。设计从"管理工厂"改为"记录协作"。
砍掉：原材料库存、半成品库存、甘特图排期、生产任务分解 → 节省1-2周开发量

功能清单

子功能优先级详细描述
工厂联系人P0添加/编辑工厂信息（名称/联系人/电话/地址/擅长品类/历史合作记录）；每个工厂可备注合作评级和注意事项
订单记录P0记录每个生产订单（关联作品+工厂+数量+单价+总价+付款方式）；手动更新订单状态（已下单/生产中/已发货/已完成）
付款记录P0记录订单付款情况（应付款/已付款/尾款/付款日期）；支持部分付款/分期付款
沟通时间线P0以时间线形式记录和工厂的沟通内容（文字备注+附件）；支持手动添加和导入（从微信截图/聊天记录）
关键事件提醒P1设置关键节点提醒（预计交货日提醒、尾款支付提醒、样品确认提醒）

说明：这个模块的价值在于创作者自己的记录——不再靠微信聊天记录找工厂信息，所有协作数据统一管理。它不是一个ERP，而是一个创作者视角的工厂协作笔记本。

10. 发布变现模块

核心变更：从CSV导出升级为一键扫码发布
系统通过Playwright浏览器自动化 + 开放平台OAuth混合方案实现一键发布：
• 用户首次使用→系统弹出平台登录页→扫码或账号密码登录
• 系统自动保存Session/OAuth Token（本地加密存储）
• 后续发布→一键选择多平台→自动填写+自动上传
• 淘宝/小红书/抖音三大平台均支持
• 无API的平台用Playwright自动化模拟（用户可见浏览器操作过程）

功能清单

子功能优先级详细描述技术实现
平台账号管理P0添加/管理各平台账号（淘宝/小红书/抖音/闲鱼等）；支持OAuth授权登录 + Cookie/Token加密存储；登录状态检测和过期提醒Playwright + 各平台OAuth SDK + SQLite加密存储
一键扫码发布P0选择作品→选择平台→系统自动填写标题/描述/标签/定价→一键发布到淘宝(商品)、小红书(笔记)、抖音(短视频/图文)等Playwright自动化 + 各平台创作开放平台API混合
批量多平台发布P0一次操作发布到多个平台；各平台内容格式自动适配；发布状态实时反馈（成功/失败/审核中）异步任务队列 + 各平台适配器
产品信息AI生成P0基于作品内容自动生成标题、描述、标签、定价建议；支持多风格（小红书风/淘宝风/抖音风）本地LLM(Ollama)或任意LLM API

15.6 关键开发最佳实践

架构原则

原则说明
模块化设计每个业务模块完全独立，可以单独开发、测试、部署
插件化第三方服务存证平台、搜索引擎等通过适配器模式接入，新增平台只需添加适配器
离线优先核心功能（作品管理、本地预览、哈希计算、数据管理）完全离线可工作
渐进式加载前端按路由懒加载，初始加载体积控制在500KB以内
文件操作安全所有文件操作加锁，防止并发问题

性能考虑

· 缩略图异步生成，首屏不等待 · 大型音视频文件流式预览，不一次性加载到内存 · 数据库定期VACUUM（SQLite需要手动回收空间） · 作品列表支持虚拟滚动，几百个文件也不卡顿 · 文件哈希计算后台线程执行，不阻塞UI

安全考虑

· 所有API密钥存储在本地配置文件中，不上传Git · 配置模板(secrets.yaml.example)放在仓库中，实际文件被.gitignore忽略 · 本地应用不需要登录密码（单机使用），但可选的PIN码锁定 · 所有第三方API调用通过HTTPS · 用户自行管理API token，系统不保存敏感第三方凭证

移植性与备份

· 所有数据（数据库+文件）都在 ~/.oristudio/ 目录下 · 备份 = 复制该目录到一个U盘/云盘 · 恢复 = 将备份目录复制到新电脑的对应位置 · Docker部署: docker run -v /host/data:/app/data oristudio · 一键迁移脚本: oristudio-migrate.sh / oristudio-migrate.ps1
