## 一、核心定位修订

### 1.1 定位变化

| 维度 | v3 原设计 | v3 修订后 |
|------|-----------|-----------|
| 目标用户 | 6类创作者全场景覆盖 | **v1 聚焦插画师/AIGC艺术家** |
| 产品定位 | 全IP创作场景一站式工具 | **插画师全链路助手** (作品管理→存证→产品化→分发→收入) |
| 能力边界 | 暗示6类全部支持 | **诚实声明: v1 仅深度支持插画师** |
| 其他类型 | 被"硬塞进来" | **基础导入支持 + 明确规划路线图** |

### 1.2 核心决策

**提出者**: 架构师 (P0-1)。6位创作者全部确认"系统是为插画师设计的, 其他类型被硬塞进来"。

**业务决策**: v1 做深做透插画师/AIGC艺术家的完整工作流，而不是6类都做到"可用但不满意"。

### 1.3 v1 聚焦插画师 — 边界定义

**v1 包含 (插画师完整支持)**:
- 作品管理: 图片格式导入 (PNG/JPG/PSD/TIFF/WebP), 系列/版本管理
- 存证确权: SHA-256哈希, 多平台锚定, PDF证书
- 侵权监测: 以图搜图 (百度识图/Google Vision)
- IP登记: 著作权/商标/外观设计指引, 费用计算
- 商业转化: 产品设计器, Canvas叠加预览, POD渠道管理
- 内容分发: 社交媒体平台 (小红书/站酷/B站/微博/Instagram), AI文案
- 收入追踪: 多渠道收入汇总, 趋势分析

**v1 不包含 (标记"规划中")**:
- 视频工作流: 工程文件项目包支持、视频指纹、Content ID
- 手工工作流: 物理原件管理、原料库存、生产批次、Etsy API上线
- 摄影工作流: RAW全流程、选片模式、水印服务、图库API同步
- 音乐工作流: ISRC管理、Split Sheets、Sample Clearance、音乐发行API
- 文字工作流: 章节层级、EPUB排版、抄袭检测

## 四、POD 渠道管理 — 诚实能力声明

**功能重定义**: "POD对接" → **"POD渠道管理"** — 创建产品 → 选POD平台 → 提供平台跳转 → **手动上架** → 记录商品URL → 追踪状态

**工作流**:
```
[产品设计完成]
  → 选择目标POD平台 (Printful/Redbubble/Printify/印鸽/Society6/Gelato)
  → 系统展示: 平台规格要求 / 设计稿适配检查结果
  → 系统提供: 一键跳转按钮 → 打开平台上传页面
  → 用户手动: 上传设计稿、填写信息、发布
  → 用户回到系统: 粘贴平台商品URL
  → 系统自动: 记录平台URL、创建监控记录
  → 定期: 提示用户更新销售状态 (手动/CSV导入)
```

**Printful Mockup API**: 优先接入 Mockup Generator API 生成照片级效果图，API不可用时自动降级到 Canvas 扁平叠加

## 六、UPL 法律合规需求

### 6.1 7项免责声明 (首次启动强制确认)

1. **不构成律师-客户关系** — 使用本软件不建立律师-客户特权关系
2. **不构成法律建议** — IP登记指引/分类推荐仅供参考
3. **不保证注册成功** — 注册结果取决于官方审查
4. **POD平台IP条款警告** — 上传前请阅读平台服务条款中的知识产权部分
5. **AI内容标注要求** — 建议按平台规则标注"AI辅助生成"
6. **侵权监测局限性** — 不能保证发现所有侵权行为
7. **司法管辖区限制** — IP登记指引仅覆盖主要辖区

### 6.2 商标分类推荐修订

**从**: "系统推荐你注册第XX类"
**改为**: "以下类别与你的作品可能相关, 请与律师确认后选择" — 返回多类可能性+置信度

### 6.3 CNIPA提交前律师审核步骤 (不可绕过)

用户必须选择: (A) 我已咨询律师 (B) 我需要找律师 (C) 我自愿跳过 [二次确认+勾选风险声明]
