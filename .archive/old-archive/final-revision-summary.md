## 一、项目概述

### 1.1 研讨目标

本次5阶段协同设计研讨针对 OriStudio v3 的全套设计文档，执行了一次从问题发现到方案落地的完整闭环：

1. **Stage 1 (PM启动)**: 基于9角色独立评估报告，产出30项结构化问题清单 + 46项任务分配矩阵 + 8个关键决策点
2. **Stage 2 (3角色并行)**: 业务专家/程序员/设计师分别产出需求文档、技术可行性报告、UX设计规范
3. **Stage 3 (架构师汇总)**: 更新全部14个设计文档 + 3个HTML架构图 + 重写实施计划
4. **Stage 4 (交叉复核)**: 程序员/设计师/业务专家三轮复核，累计修复40项问题
5. **Stage 5 (本报告)**: PM最终汇总，输出本次研讨的完整结论

### 1.2 核心成果

**一句话总结**: 将 OriStudio v3 从"试图覆盖6类创作者但每种都不深入"，重构为"v1聚焦插画师/AIGC艺术家做到极致"的诚实产品，消除法律合规风险，建立可交付的实施路径。

## 二、核心决策清单

以下8个关键决策点已在 Stage 2-3 完成评审确认：

| ID | 决策 | 结论 |
|----|------|------|
| **KD-1** | v1聚焦策略 | v1 聚焦**插画师/AIGC艺术家**单一类型，其他5类标注"规划中"。数据模型预留扩展点但不实现专属工作流 |
| **KD-2** | POD能力边界 | 重命名"POD对接"为"POD渠道管理"。Printful接入Product Catalog API(读)+创建草稿(写)，其余平台手动链接记录。文档诚实标注能力边界 |
| **KD-3** | Canvas预览方案 | 三层降级: A) Canvas扁平叠加(默认) B) Printful Mockup API(增强P1) C) PSD模板(P2规划)。UI标注"平面效果预览，非真实产品照片" |
| **KD-4** | UPL法律合规 | 7项免责声明嵌入全系统。尼斯分类返回多推荐+置信度(最少3条)。CNIPA提交强制律师审核步骤(不可绕过)。移除可注册性评估/成功率评分 |
| **KD-5** | v1修订范围 | P0全部解决(6项)。P1至少完成数据模型补齐+文档匹配+批量导入(3项)。P2纳入backlog。总计30项问题，v1门槛=6项P0 |
| **KD-6** | 非插画类型UI处理 | 显示但标注"规划中"。插画师卡片视觉突出(大尺寸+渐变边框+推荐徽章)。透明诚实优先于隐藏 |
| **KD-7** | 实施计划重写 | 从6 Phase/6天改为4 Phase/13-14天，优先合规修复而非功能构建 |
| **KD-8** | 新手体验投入 | v1最小可用: 3步Onboarding向导 + 10空状态 + 9术语替换 + 5核心Tooltip。完整设计系统升级推迟到v1.5 |
