**业务架构四层说明：**

| 层次 | 职责 | 关键要素 |
|------|------|---------|
| **用户层** | 系统服务的创作者群体 | 6 类创作者类型，AI 渗透所有类型 |
| **业务价值链** | 从创作前到商业变现的完整生命周期 | 4 阶段：风险预警 → 存证确权 → 监测维权 → 商业撮合 |
| **能力层** | 支撑业务价值的 AI Agent 编排 + 第三方服务集成 | 3 大 Agent (RiskWarning/Enforcement/InnocenceProof)，MCP 协议连接外部智能体 |
| **生态层** | 不重复造轮子 — 对接权威第三方 | 区块链、商标库、以图搜图、POD 平台、社交分发 |

**技术架构七层说明：**

| 层次 | 技术选型 | 职责 |
|------|---------|------|
| **表现层** | Vue 3 + Naive UI + Tailwind CSS | 用户界面、向导式交互、空状态引导 |
| **应用层-Router** | FastAPI 21 路由文件 | REST API 端点 (334+) |
| **应用层-Service** | Python 业务逻辑 | 跨模块业务编排 |
| **应用层-Agent** | Celery 异步任务 | AI Agent 编排，对外表现为自动化 |
| **集成层** | Gateway ABC 模式 | 14+ 外部 API 适配器，统一抽象 |
| **数据层** | SQLite + SQLAlchemy 2.0 | 102 模型类，WAL 模式 |
| **基础设施** | Celery/WebSocket/Redis | 定时任务、实时通知、消息队列 |
