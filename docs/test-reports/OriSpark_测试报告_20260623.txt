## 一、基础定义与测试策略
### 1.1 问题严重等级定义
| 等级 | 标识 | 判定规则 | 整改优先级 |
|------|------|----------|------------|
| P0 致命 | 阻断核心业务、违反法律法规、业务逻辑完全倒置、产品无法商用 | 紧急修复（上线前必须改完） |
| P1 严重 | 核心功能缺失、校验逻辑失效、操作路径无效、关键信息错误，大幅影响业务使用 | 下一迭代版本优先整改 |
| P2 一般 | 功能可用但设计不合理、文案错误、入口冗余、逻辑漏洞、信息不完整 | 常规迭代优化 |
| P3 轻微优化 | UI排版、hover提示、动效、文案细节、引导缺失类体验问题，不阻碍业务运行 | 排期按需优化 |

### 1.2 测试方法
1. **全节点遍历法**：逐个点击每一个菜单、按钮、标签、跳转链接、关闭控件，全覆盖无遗漏；
2. **业务场景法**：模拟真实创作者完整运营链路：创作素材入库→版权确权登记→侵权维权→内容分发→商业变现→数据复盘校验逻辑合理性；
3. **等价类+边界值测试**：针对上传大小、文件格式、字数、费用数值、办理周期做边界校验；
4. **异常场景测试**：无前置数据直接访问下游页面、重复操作、空操作、非法输入场景验证；
5. **UI/UX规范性测试**：图标统一性、层级结构、弹窗逻辑、导航、状态反馈、新手引导校验。

## 二、逐条详细测试用例（预置条件+操作步骤+预期结果+实际结果+问题+整改）
### 模块一：左侧侧边导航栏 逐控件全量测试
#### 用例1：侧边栏收起/展开箭头「◀」按钮测试
- 预置条件：系统正常加载，侧边导航默认展开状态
- 测试路径：页面左上角 ◀ 箭头控件
- 操作步骤：
  1. 鼠标悬浮箭头查看提示；
  2. 点击箭头执行操作；
  3. 再次点击切换状态；
  4. 刷新页面查看状态是否留存
- 预期结果：
  1. Hover提示：收起/展开侧边导航栏；
  2. 点击收起侧边栏，箭头变为▶；再次点击展开恢复◀；
  3. 切换有平滑过渡动画；
  4. 布局状态浏览器本地缓存，刷新不重置
- 实际结果：
  1. 无Hover文字提示；
  2. 点击后箭头方向不变，无视觉变化；
  3. 无切换动画；
  4. 刷新页面布局状态重置
- 问题等级：P3
- 问题描述：侧边栏开关交互缺失引导、状态反馈失效、配置不持久，易用性差
- 整改建议：
  1. 增加Hover气泡提示文案；
  2. 点击双向切换箭头符号，增加300ms平滑过渡；
  3. 收起/展开状态存入LocalStorage持久化；
  4. 收起模式支持鼠标悬浮临时展开导航

#### 用例2：一级菜单【概览】点击跳转测试
- 预置条件：当前停留在IP登记页面
- 测试路径：侧边栏 → 概览
- 操作步骤：点击「概览」菜单
- 预期结果：正常跳转系统概览首页，面包屑同步更新
- 实际结果：可点击跳转，但页面无面包屑路径提示
- 问题等级：P2
- 问题描述：页面跳转后无位置导航，多层级浏览后用户迷失页面位置
- 整改建议：全站二级及以上页面顶部增加面包屑导航：`当前位置：概览`，支持点击回退

#### 用例3：侧边栏分组命名缺陷：经营管理分组+子菜单同名测试
- 预置条件：侧边栏完整展示
- 测试路径：侧边栏 → 经营管理（分组标题）→ 子菜单「经营管理」
- 操作步骤：观察层级命名，点击子菜单进入页面
- 预期结果：分组名称与下属子菜单名称必须差异化，层级语义清晰
- 实际结果：分组标题叫「经营管理」，内部子菜单同样命名「经营管理」，父子同名，用户无法区分分类容器与功能页面
- 问题等级：P1
- 问题描述：菜单层级设计不规范，极易造成用户理解混淆，后期迭代菜单管理混乱
- 整改建议（二选一）：
  方案1：分组重命名为【经营数据与资产管理】；
  方案2：子菜单修改为【经营数据总览】；
  落地内部规范：禁止上下级菜单同名

#### 用例4：核心业务菜单「创意资产（角标10）」点击测试
- 预置条件：系统存在10条创意资产数据
- 测试路径：侧边栏 → 核心业务 → 创意资产
- 操作步骤：
  1. 鼠标悬浮菜单查看说明；
  2. 点击进入页面；
  3. 校验角标数字含义是否有解释
- 预期结果：
  1. Hover气泡说明：创意资产：原创作品素材存储仓库，IP运营起始入口；角标为当前有效作品总数；
  2. 正常跳转创意资产列表页；
- 实际结果：无悬浮说明，仅展示数字10，新用户不清楚角标含义、模块定位
- 问题等级：P3
- 问题描述：模块无释义，业务定位模糊
- 整改建议：全部侧边业务菜单增加Hover功能说明，角标补充释义逻辑

#### 用例5：核心业务菜单「权利保护（角标0）」访问前置逻辑测试（关键业务逻辑用例）
- 预置条件：**未上传任何创意资产、未完成IP登记确权**（真实业务前置条件）
- 测试路径：侧边栏 → 核心业务 → 权利保护
- 操作步骤：直接点击进入页面
- 预期结果：
  1. 前置拦截弹窗：暂无已确权IP作品，维权操作需要先上传作品并完成IP登记；
  2. 弹窗提供一键跳转【创意资产】入口；
  3. 页面禁止直接进入，杜绝无效操作
- 实际结果：无任何拦截、无弹窗提示，可直接进入权利保护空白页面
- 问题等级：P1
- 问题描述：违背真实知识产权业务逻辑：未确权作品无法律维权基础，系统缺少上下游业务依赖校验，产生大量无效操作
- 整改建议：
  1. 给权利保护、商业转化、内容分发全部增加前置访问校验；
  2. 无确权资产时拦截访问并引导上游流程；
  3. 空白页面配置空状态引导图+操作指引文案

### 模块三：全局横向业务流转导航条（**核心P0致命业务逻辑测试**）
- 预置条件：模拟真实创作者完整IP运营场景
- 测试路径：工作台顶部流程：`创意资产 → 权利保护 → IP登记 → 商业转化 → 内容分发 → 经营管理`
- 操作步骤：梳理知识产权国家法定真实业务流程对比校验
- 真实合规业务顺序（著作权/IP运营行业标准）：
  **创意资产（作品上传入库）→ IP登记（确权拿权属凭证）→ 权利保护（侵权监测、维权投诉）→ 内容分发（全网发布）→ 商业转化（授权变现）→ 经营管理（数据对账复盘）**
- 实际系统流程：先维权、后确权，顺序完全颠倒
- 问题等级：**P0 致命**
- 问题详细描述：
  1. 法律层面：作品未完成登记确权，无权属证明，不具备发起维权的法律基础，流程完全不符合著作权法实操逻辑；
  2. 业务层面：整条链路业务闭环失效，产品核心IP运营模式不成立，无法商业化交付；
  3. 流程节点不可点击跳转，无进度状态可视化，无法体现项目推进情况
- 整改建议：
  1. 紧急调整流程节点顺序至行业合规顺序；
  2. 改造为步骤进度条，增加序号、三色状态（未开始/进行中/已完成）；
  3. 节点支持点击快速跳转对应页面，展示单个IP项目进度看板
